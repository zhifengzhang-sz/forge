# v1 — Scale to All Four Domains, 5000 Pairs

This document is the handoff for a **fresh Claude Code session** to execute v1.
All facts needed are captured here or referenced via committed files. For
prior context see `v0/decision.md`, `v0.6/decision.md`, and `v0.7/decision.md`.

## Why v1 exists

v0.7 validated the recipe on three domains (XState, FP, RX). Every
predeclared threshold cleared: XState 4.10, FP 4.40, Reactive 4.80, tool
calls 3/3. Winner was r=64 with 4.10 domain average.

Unaddressed at v0.7:
- **Event Sourcing stayed at 3.00–3.10** because we never trained on it.
  Same story across v0, v0.6, and v0.7 — ES has been the untouched domain
  throughout.
- **Scale plateaued at ~1000 pairs.** The ~1k LoRA floor cited in v0.6's
  plan was treated as a floor, not as a target. v1 tests whether 5000
  pairs pushes the trained domains past 4.5 toward the Opus ceiling at 4.95.

v1 is the scale-up run that adds ES to the mix and tests whether the
recipe continues working at 5× volume.

## Predeclared outcomes (same structure as v0.7)

| Condition | Conclusion |
|---|---|
| XState ≥ 4.5 AND FP ≥ 4.0 AND Reactive ≥ 4.60 AND ES ≥ 3.8 | **Premise exceeded. Ship as daily driver (see Phase I).** |
| All domains trained ≥ 3.5 but below the above thresholds | Solid v1. Consider quantization / prompt engineering rather than more data. |
| Any trained domain (XState/FP/RX) drops ≥ 0.3 below v0.7 baseline | **Automated halt** (combine_v1.py regression gate, Phase G). Data mix investigation required. |
| ES stays below 3.5 despite training | ES-specific patterns may need different approach (different seed style, more complex examples). |

RX threshold tightened from 4.5 to 4.60 (v0.7 - 0.2) because v1 reduces RX's
proportion from 23% → 12%; a drop between 4.5 and 4.79 would clear the old
threshold while masking a real regression.

Tool calls ≥ 2/3 required in every branch. Uncertainty bands (two-grader)
required as in v0.7.

## Plan

### Phase A — Seeds

Extend `v1/seeds/` from v0.7 seeds:
1. Copy `v0.7/seeds/patterns.xstate.md`, `patterns.fp.md`, `patterns.rx.md`,
   `phrasings.md`, `reference_examples.xstate.jsonl`, `anchors.jsonl`.
2. Write `v1/seeds/patterns.es.md` — **new**. ~15 ES patterns covering
   the bullets in the scope table above. Each entry matches the style
   of v0.7/seeds/patterns.fp.md (one-line summary + key APIs + must-haves).
3. Write `v1/seeds/reference_examples.es.jsonl` — **25–30 real ES examples
   mechanically extracted from `repos/EventSourcing.NodeJS/`** (samples/
   foundations, samples/snapshots, samples/hotelManagement — confirmed to
   contain Oskar-style `when + aggregateStream` AND Decider-style
   `evolve/decide` patterns). Same extraction approach that worked for
   v0.6 XState (and beat hand-curation in v0).
4. Expand `anchors.jsonl` to **30 unique anchors**: keep the 12 original
   as a subset + author 18 new anchors covering additional capability
   surface (file ops variants, grep/find variants, git status/log/diff,
   bash/python helpers, test runners, tool dispatch edge cases).
5. **Contamination audit (new, pre-flight):** before Phase C, diff the
   extracted ES `reference_examples.es.jsonl` token bag against the eval
   ES prompts (es-01..es-05). Require < 50% token overlap per prompt. If
   any eval prompt's signature (function name, domain concept) appears
   verbatim in a seed, either rewrite the eval prompt or swap that seed.
   Rationale: eval and extraction both draw from Oskar's canonical
   surface; need explicit separation to grade generalization, not
   memorization.

### Phase B — Verifier (per-domain, extended to ES)

**Extend `v0.7/verify.py` in-place.** Add `"es"` to `DOMAINS`, extend
`_IDIOMS`, `_package_json_for()`, `_marker_package_for_domain()`.
`v1/verify.py` is a thin wrapper with v1-specific paths. DO NOT fork
(saves 723 lines of duplication, preserves one canonical verifier).

Re-run `v0.7/test_verify.py` after extension to confirm XState/FP/RX
behavior unchanged. Add ES test fixtures: Oskar-style (when + aggregateStream),
Decider-style (evolve + decide), Effect+ES wrap, xstate-leakage (must fail),
setup()-leakage (must fail).

| Gate | ES |
|---|---|
| Compile | `tsc --strict --noEmit` with deps: `@types/node`. Keep the dependency set minimal. |
| Idiom MUST-MATCH (any of, 18 tokens) | `evolve\(`, `decide\(`, `\.decide\(`, `Decider`, `CommandHandler`, `EventStore`, `EventStoreDB`, `Snapshot`, `expectedRevision`, `appendToStream`, `readStream`, `readFromStream`, `when\(`, `aggregateStream\(`, `\breduce<`, `\bprojection`, `\baggregate` |
| Idiom MUST-NOT-MATCH | `from ['"]xstate['"]`, `setup\(` (shape guards). **Effect.gen removed** — Effect-wrapped event store access is a valid ES idiom. |
| Length | 150–6000 chars |

XState / FP / RX gates: unchanged from v0.7. Test fixtures re-run to
confirm no regression from the ES extension.

### Phase D — Verify + merge

Same as v0.7. Per-domain tsc gate runs, combined output written to
`v1/data/synth.verified.jsonl` with domain field per record.

### Phase E — Train

v1/train.py = fork of v0.7/train.py. Defaults:
- Rank **r=64** (locked from v0.7 winner; r=128 deferred to v1.1 in TODOS.md).
- Epochs **3** (locked).
- LR **1e-4** (locked).
- Anchor ratio **12%** (30 unique × 20 reps).
- Seed 42.

Single training arm for v1 (r=64 only). r=128 A/B is in TODOS.md for v1.1 —
running as single-variable ablation after v1 signal is in keeps experiments
clean.

Same pre-flight checks:
- `systemctl is-active nvidia-pl.service` → active
- `nvidia-smi --query-gpu=power.limit --format=csv,noheader` → 500 W
- 30 s sleep between `save_pretrained_gguf` and `ollama create`.

Training wall time estimate (based on v0.7: 15 min per arm at 1053 records):
5000 records × 3 epochs / batch size 8 ≈ 1875 steps. Assuming 0.5 s/step
on the 5090 → ~15–20 min training + 7 min GGUF conversion = ~25 min per arm.

### Phase F — Evaluate

Unchanged: `v0/run_eval.py --model ts-forge-v1 --arm v1` and
`v0/tool_call_smoke.py --model ts-forge-v1 --arm v1`. Keep eval set at
24 prompts for apples-to-apples with v0 / v0.6 / v0.7.

### Phase G — Grade (hard-isolated two-grader + regression gate)

- Grader A: scores v1 r=64. Also re-grades **v0.7-r64 ES** in the same
  session for calibration anchoring (prevents drift where trained ES
  output shifts the reference distribution). Writes
  `v0/results/v1.graded.grader_A.json`.
- Grader B: explicitly forbidden to read any `v0.7*.graded.*` or
  `v1.graded.*` file. Same v0.7-r64 ES re-grade in-session. Writes
  `v0/results/v1.graded.grader_B.json`.
- Combine via `v0/grading/combine_v1.py`. Emits:
  - Per-domain means + two-grader uncertainty bands
  - Per-anchor Cap breakdown: `Cap_old12` (comparable to v0.7 4.25) and
    `Cap_new18` (new surface)
  - **Automated regression gate**: if any trained domain
    (XState/FP/RX) drops ≥ 0.3 below v0.7-r64 baseline, exit non-zero and
    halt the pipeline. Decision.md NOT written.
- If any domain falls within ±0.3 of a predeclared threshold, dispatch
  one ad-hoc 3rd grader for that domain only (hard-isolated) as
  tiebreaker.

Write `v1/decision.md` matching the v0.7 format. Apply predeclared outcome

## Plan-eng-review decisions (resolved 2026-04-19)

| Q | Decision |
|---|---|
| 1. ~5000 scale? | **Yes, 5000.** 2026 LoRA literature places 5000 exactly at the diminishing-returns cliff for domain specialization. 3000 is the shortcut option. |
| 2. Proportion 20/24/20/30/5? | **Rebalanced** to 24/24/12/28/12 — cut RX (near ceiling), kept ES high but with canary gate, expanded anchors (see Q4). |
| 3. r=128 A/B? | **No.** Lock r=64. r=128 deferred to v1.1 as single-variable ablation (TODOS.md). |
| 4. Anchor memorization? | **Real risk.** Expanded to 30 unique × 20 reps (12 original + 18 new). Per-anchor grading breakdown isolates new-anchor drift. |
| 5. Eval expand? | **Defer to v1.1.** v1 = scale+ES experiment. Don't change yardstick and experiment simultaneously. n=5 caveat acknowledged in decision.md. |
| 6. Grader pool? | **Two graders + ad-hoc 3rd on ±0.3 borderline.** v0.7 disagreement was 0.25, 3rd grader baseline is overkill. |
| 7. ES seed process? | **Extract from repos/EventSourcing.NodeJS** (25–30 examples) + **hand-author 15 patterns.md entries.** Mirrors v0.6's winning approach. |
